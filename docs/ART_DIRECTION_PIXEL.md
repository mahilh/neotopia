## WHY PIXEL ART (not painterly, not photorealistic)

  Reads instantly at card size (120px wide) — pixel clusters are MORE readable than
  complex paintings at small scale. Cannot look like AI slop — pixel art has hard
  structural constraints that enforce consistency. No other consciousness civilization
  game uses pixel art cards. Unique. Ancient wisdom in digital form.

  The feeling: Stardew Valley meets Chrono Trigger meets ancient mystery school.
  The era: SNES / 16-bit RPG · the golden age of isometric pixel architecture.
  The energy: esoteric solarpunk · ancient + digital · warm jewel-tone palette.

## MASTER PIXEL ART STYLE STRING

Append to every card prompt:
  '16-bit pixel art, SNES RPG style, isometric view, crisp pixel grid, limited color palette
   of 8-16 colors, no anti-aliasing, flat color areas within pixels, visible pixel edges,
   fantasy game sprite, solarpunk jewel-tone colors, teal blue green amber gold palette,
   square 1:1 format, pixel art only, NOT painterly, NOT photorealistic, NOT 3D render'

## CHATGPT PROMPT TEMPLATE

For each card:
  '16-bit pixel art of [BUILDING NAME AND DESCRIPTION].
   SNES RPG isometric view. Limited pixel palette of teal, amber, gold, green.
   Crisp pixel edges. No gradients. No anti-aliasing. Flat pixel colors.
   Esoteric solarpunk fantasy sprite. Sacred geometry elements visible.
   Square format 1:1.'

## THE TWO-LAYER CARD SYSTEM

  Layer 1 (CODE · already built): CardFrame.jsx
    Ancient esoteric frame: dark obsidian background, element-color borders,
    sacred geometry overlay, serif Roman numerals, ancient corner symbols.
    This layer is done. Don't change it.

  Layer 2 (ART · this document): pixel art illustration inside the frame
    16-bit isometric pixel sprite of the building.
    Saved to: public/art/cards/[card-id].png
    CardFrame loads it automatically when the PNG exists.

  Combined effect: ancient mystical card frame containing a pixelated vision
  of the future. Like an ancient data crystal showing pixel-art prophecies.

## ART DELIVERY WORKFLOW

  1. Generate each card in ChatGPT (paste prompt exactly)
  2. Download the image
  3. Save to: ~/NeoTopia/public/art/cards/[exact-filename].png
  4. Commit: git add public/art/cards/ && git commit -m 'art: batch [N] pixel art cards'
  5. CardFrame.jsx auto-loads the art — no code change needed

## QUALITY CHECK (before saving)

  PASS if:
    Visible pixel grid · limited color palette · clean edges · isometric view
    Single recognizable building · solarpunk color palette

  FAIL if:
    Gradients · anti-aliasing · painterly brushstrokes · photorealistic
    Too many colors · messy detail · no recognizable building

  If fail: regenerate with same prompt. ChatGPT pixel art is inconsistent.
  2nd attempt usually cleaner. Accept the 3rd attempt regardless.
